Контент веб-сайта

Содержимое сайта разделяется на статический и динамический контент.

Статический контент — это, например, HTML-файлы, файлы изображений, CSS-файлы и другие файлы, которые просто лежат на диске. Директива DocumentRoot указывает где в вашей файловой системе, размещаються эти файлы.

Обычно, когда запрашивается каталог, без указания имени файла, то будет отдан документ с именем index.html. Например, если для директивы DocumentRoot установлено значение /var/www/html и приходит запрос на адрес http://www.example.com/work/, то файл расположенный по пути /var/www/html/work/index.html будет отдан клиенту.

Журнал

Расположение журнала ошибок задаётся директивой ErrorLog, которая может быть установлена глобально или для каждого виртуального хоста.

Установка Apache

Установка Apache ~]$ sudo yum install httpd.

После установки появлятеся служба httpd. Смотрим статус службы 

                                
                                    ~]$ systemctl status httpd
○ httpd.service - The Apache HTTP Server
     Loaded: loaded (/usr/lib/systemd/system/httpd.service; disabled; vendor preset: disabled)
     Active: inactive (dead)
       Docs: man:httpd.service(8)
По началу служба отключена и у нее не настроен автоматческий запуск. Вывод disable означает автоматический запуск выключен. Active: inactive (dead) не запущена.

Запускаем: ~]$ sudo systemctl start httpd
и включаем атозапуск: ~]$ sudo systemctl enable httpd
Все включилось. Смотрим сататус: ~]$ systemctl status httpd

Перезагрузить web-сервер Apache: ~]$ sudo systemctl restart httpd По мере работы со службой мы будем управлять нашим веб-сервером: менять его кофигурацию, добавлять дополнительных модулей. После каждой такой операции веб-сервер необходимо перезаргужать. При restart соединение с клиентом обрываеться

Перечитать кофигурацию: ~]$ sudo systemctl reload httpd При reload соединение с клиентом не оборвется. Но релоад не всегда применим.

.

Смортим порт:~]$ sudo ss -tupln | grep httpd Apache по умолчанию слушает 80 порт.

Смотрим конфигурацию файрвола: ]$ sudo firewall-cmd --list-all В кофигурации файрвола, мы увидим что в наборе сервисов открытых для доступа извне, отсутствует httpd: services: dhcpv6-client samba-client ssh Его надо туда добавить.

Добавить Apach в активную конфигурацию файрфола на постоянной основе ~]$ sudo firewall-cmd --add-service=http --permanent чтобы к нему могли подкючаться клиенты. success - значит добавили!

~]$ curl http://192.168.0.12 | head Можно проверять что сервер работает. Можно проверять как на самом сервере так и с удаленной машины. Ну и через браузер соответственно можно глянуть.

Как управлять web-сервером Apache

Для управления web-сервером обычно используються конфигурационные файлы. Каждый раз запуске web-сервер считывает из них кофигурацию и применяет ее к себе. У Apache не очень сложная структуа кофигурационных файлов.

Конфигурационные файлы:

  • Основной конфигурационный файл:
    • /etc/httpd/conf/httpd.conf
    • В нем прописываються все основные директивы,
    • + там указыватся загрузка двух дополнительных директив
  • Вспомагательный каталог для кофигурационных файлов:
    • /etc/httpd/conf.d
    • Все файлы с раширением .conf находящиеся в этом каталоге, тоже считываются и применяться к основоной конфигурации.
  • Вспомагательный каталог для конфигурационных модулей:
    • /etc/httpd/conf.modules.d/
    • размещаються конфиг.файлы от модулей.

Проверить конфиг.файлы: ~]$ sudo apachectl configtest проверить на ошибки конфиг.файлы. Это если что то будите менять в конфиг.файлах то эта команда проверит правильно ли вы внесли корректировку и все ли в порядке с ними. Прежде чем перезагрузить Apache или перечитать его конфигурацию, нужно это проверить.

Что внутри глвного конфиг.файла

Ну есесено все что начинеться с # это коментарии. Далее. ServerRoot "/etc/httpd" - корневой котолог для конфигурации.

Listen ip-адрес и порт который прослушивать. У нас прослушивает 80 порт а ip-адрес закоментирован, значит он прослушивает все доступные ip-адреса

Далее идут подключения конфиг.файлов Include conf.modules.d/*.conf - это подкючение конфигов для модулей.

User и Group - пользователь и группа от имени которых работает Apache.

Дальше идет кофигурация котологов <Directory "/var/www>... </Directory> Посмотрим на конфигурацию каталога /var/www а так же на /var/www/html - это катлог в который можно сразу после установки web-сервера начитать загружать каки-либо данные и отсюда эти данные будут считывать, ну т.е можно уже размещать тут сайт.

В самом конце главного конфига есть включение опциональных файлов IncludeOptional conf.d/*.conf т.е если вы не хотите хранить какую то конфигурация в основном конфиге, можно храть ее там отдельном файле..

Запретим просмотр каталогов

Модуль Apache рисует структуру каталогов в браузере

Ну например если у вас есть в корне сайта каталог /css то по этому url: http://linuxgic.ru/css Apache нарисует вам структуру каталогов, а если укажите путь до файла http://linuxgik.ru/css/style.css то пакежет и файл. В конфигурации каталага /var/www/html в Options удалим параметр Indexes - перечитаем фалй конфигурации, и тогда Apache не будет отрисовавать структуру каталогов. За эту отрисовку отвечает определенный модуль, и можно настроить как он будет отрисовывать.
