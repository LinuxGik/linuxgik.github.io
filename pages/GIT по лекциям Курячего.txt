GIT по лекциям Курячего 

GIT распределенная система контроля версий 
Один из главных приципов: работа происходит по технологии пул. 
У каждого участника процесса свой собственный публичный реп, тут он публикует свои изменения, так же у каждого есть свой локальный реп где он делает все изменения. т.е процесс пуш и коомит разделены. коммитов может быть много
(каждое измение оформляется как отдельый коммит) а финальный пуш один которыйй пушит сразу все коммиты и пушит он их не в общию кучу а в лично ваш публик.реп.
Когда возникает необходимость слить все в одно, специально выделенный человек, берет те репы которые посчитает нужным и все их мержит в один.
 
Очень развито управление ветками
Так же очень Аксесс контрол листы, управление доступом
Зачем на репозиторий.
Если мы хотим внести какие то изменения, мы с ним синхронизируемся (скачиваем файлы на свой компьютер) =>  редактирование => коммитим (какая задача решалась и каким способом) => публикуем. 


git pull синхронизация репозитория 
git status покажет что есть в рабочей копии
