rsync

Должна быть установлена на всех машинах.

В rsync всегда, сначала указываем источник, а потом место назначения, т.е сначало что копируем, а потом куда копируем.
Всегда используйте параметр -nv перед прогоном. rsync может натворить огромную головную боль если что то не так сделать.

Завершающий слеш /

Источник без завершающего слеша скопирует сам каталог /path/dir в удаленный каталог /dest_dir :
local]$ rsync -a /path/dir user@host:dest_dir

Путь к каталогу источнику с косой чертой в конце скопирует не сам каталог /path/dir, а все содержимое в удаленный каталог dest_dir:
local]$ rsync -a path/dir/ user@host:dest_dir

Небольшая инструкция по синхронизации

1. Сихронизируем локальный каталог с удаленным сервером:
local]$ rsync -a -e "ssh -p 7967" html/ user@host:html/
прим: -e "ssh -p 7967" указывает коннектиться по определенному порту.

2. На удаленной машине синхронизируем каталоги:
remote]$ rsync -a html/ /var/www/html/

Исключить каталог из операции переноса

local]$ rsync -a -e "ssh -p 7967" --exclude 'res/header' html/ user@host:html/

Команда scp

scp - копирует файлы между хостами в Сети. Работает подобно командеcp

Примеры:

  • Файл с удаленного хоса к себе в текущию дерикторию - ~]$ scp user@host:path_file path_copy_past
  • Каталог c удаленного хоста через другой порт - ~]$ scp -r -P 2871 user@host:path_file path_copy_past

Внимание: для scp порт указываеться с большой P

Инструкция по настройке сервера

1. Создаем пользователя с правами sudo

  • ~]# useradd new_username - добавляем пользователя
  • ~]# passwd new_username - задаем ему пароль
  • ~]# usermod -aG group_name1,group_name2,... user_name - добавляем пользователя в группу wheel

Выходим из сессии root и заходим под новым пользователем.

2. Настройка SSH

  • ~] $ssh-keygen - на клиенте генерируем пару ключей.
  • ~] $ssh-copy-id user@host - копируем открытый ключ на удаленный сервер.
  • Отключаем аутентификацию по паролю (в файле /etc/ssh/sshd_config) - PasswordAuthentication no
  • Запрещаем логиниться от root (в файле /etc/ssh/sshd_config) - PermitRootLogin no
  • Меняем порт (в файле /etc/ssh/sshd_config) - Port #номер
  • Перезагружаем сервер ~]# systemct reload sshd
  • Далее нужно обновить политику SeLinux чтобы разрешить использование этого порта:
    • ~]# yum install policycoreutils-python-utils
    • ~]# semanage port -a -t ssh_port_t -p tcp 2233
    • ~]# semanage port -l | grep ssh
  • Открываем порт в firewall
    • # firewall-cmd --add-port 2233/tcp
    • # firewall-cmd --runtime-to-permanent
    • # firewall-cmd --reload

local]$ ssh -p 2233 username@ip_address

или можно ставить в конце local]$ssh username@ip_address -p 2233

3. Установка web-серевера Apache

  • ~]# yum install httpd
  • ~]$ sudo systemctl enable httpd --now
  • ~]# firewall-cmd --zone=public --add-service=http --permanent
  • ~]# firewall-cmd --reload

Как использовать sss-ключи

Команда ~] $ssh-keygen - помещает сгенерированные ключи id_rsa и id_rsa.pub в каталог ~/.ssh. Копируем эту пару на флешку.

Чтобы ключи работали на другой машине, создаем на новой машине каталог ~/.ssh и копируем с флешки туда свою пару ключей. Обязательно выставляем этой паре ключей нужные права ~]$ chmod 600 id_rsa id_rsa.pub

Добавит открытый ключ на удаленный хост:
~]$ ssh-copy-id user_name@host_name
Открытые ключи заисываются в файл ~/.ssh/authorized_keys

Знать

Ввод и вывод с помощью оболочки

stdin- стандартный ввод

stdout - стандартный вывод

stderr - стадартаня ошибка

~]$ ls > file.txt - отправит вывод в file. Создаст file, если file уже есть затрет данные в нем.

~]$ ls >> file.txt - вывод в файл без затирания

~]$ set -C - включить параметр оболочки предотварщающий затирание (+С отключить).

~]$ head /proc/cpuinfo | tr a-z A-Z - отправить стандартный вывод на стандартый вход другой комаде

~]$ ls /ffffff 2> err.txt - отправить ошибку в файл.

~]$ ls /ffffff 2>&1 file.txt - направит ошибку туда же куда stdin
