Зачем мне всё это надо?

Частенько (в основном если Linux стоит рядом с Windows) из файловой системы Windows перекидиваются файлы в Linux-систему, и у этих фалов не корректные права доступа. Чтобы задать им (файлам и каталогам) правильные права доступа, нужно знать umask.

Что такое umask

umask (user file creation mask) — это механизм, который определяет права доступа по умолчанию для новых файлов и каталогов при их создании.

Как работает umask

  • umask — это маска, которая "вычитается" из максимальных прав доступа, чтобы установить права по умолчанию.
  • Для файлов максимальные права: 666 (rw-rw-rw-), для каталогов — 777 (rwxrwxrwx).
  • umask задаётся в восьмеричном формате (например, 022 или 002).

Примечание: Значения 0666 и 0777 жёстко закодированы в ядре как стандартные.

Примеры:

если umask = 022 (по умолчанию во многих дистрибутивах), тогда:

  • Для файлов: 666 - 022 = 644 → -rw-r--r--
  • Для каталогов: 777 - 022 = 755 → drwxr-xr-x
