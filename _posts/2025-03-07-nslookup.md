## nslookup

Это мощный инструмент для диагностики DNS (Domain Name System), который помогает узнать, как доменные имена преобразуются в IP-адреса и наоборот.

### **Что такое `nslookup`?**
`nslookup` (от **Name Server Lookup**) — это сетевая утилита, которая позволяет:
1. Запрашивать DNS-серверы для получения информации о доменных именах и IP-адресах.
2. Проверять, как доменные имена разрешаются в IP-адреса (и наоборот).
3. Диагностировать проблемы с DNS, например, если сайт не открывается или пинг не работает.

---

### **Основное назначение `nslookup`**
1. **Разрешение доменных имен в IP-адреса**:
   Например, ты можешь узнать, какой IP-адрес соответствует домену `ya.ru`.

2. **Обратное разрешение (Reverse DNS Lookup)**:
   Ты можешь узнать, какое доменное имя соответствует определенному IP-адресу.

3. **Диагностика DNS-проблем**:
   Если сайт не открывается, `nslookup` поможет определить, связана ли проблема с DNS (например, если DNS-сервер не возвращает правильный IP-адрес).

4. **Проверка работы DNS-серверов**:
   Ты можешь указать конкретный DNS-сервер для запроса, чтобы проверить, как он обрабатывает запросы.

---

### **Как использовать `nslookup`?**

#### 1. **Базовый синтаксис**
```bash
nslookup <доменное имя или IP-адрес>
```
Пример:
```bash
nslookup ya.ru
```

#### 2. **Режим интерактивного терминала**
Если просто ввести `nslookup` без аргументов, откроется интерактивный режим, где можно выполнять несколько запросов подряд. Чтобы выйти из этого режима, введи `exit`.

Пример:
```bash
nslookup
> ya.ru
> google.com
> exit
```

#### 3. **Указание DNS-сервера**
Ты можешь указать конкретный DNS-сервер для запроса. Например:
```bash
nslookup ya.ru 8.8.8.8
```
Здесь `8.8.8.8` — это публичный DNS-сервер Google.

---

### **Примеры использования**

#### 1. **Запрос IP-адреса для домена**
```bash
nslookup ya.ru
```
Вывод:
```
Server:		127.0.0.53
Address:	127.0.0.53#53

Non-authoritative answer:
Name:	ya.ru
Address: 77.88.55.242
Name:	ya.ru
Address: 5.255.255.242
Name:	ya.ru
Address: 77.88.44.242
Name:	ya.ru
Address: 2a02:6b8::2:242
```
Здесь:
- `Server` — DNS-сервер, который обработал запрос.
- `Non-authoritative answer` — ответ от кэширующего DNS-сервера (не от authoritative сервера домена).
- `Address` — IP-адреса, связанные с доменом `ya.ru`.

#### 2. **Обратный DNS-запрос (Reverse Lookup)**
```bash
nslookup 77.88.55.242
```
Вывод:
```
242.55.88.77.in-addr.arpa	name = ya.ru.
```
Это показывает, что IP-адрес `77.88.55.242` связан с доменом `ya.ru`.

#### 3. **Запрос MX-записей (для почтовых серверов)**
```bash
nslookup -query=mx ya.ru
```
Вывод:
```
ya.ru	mail exchanger = 10 mx.yandex.ru.
```
Это показывает, что почтовые серверы для домена `ya.ru` используют `mx.yandex.ru`.

#### 4. **Запрос NS-записей (DNS-серверы домена)**
```bash
nslookup -query=ns ya.ru
```
Вывод:
```
ya.ru	nameserver = ns1.yandex.ru.
ya.ru	nameserver = ns2.yandex.ru.
```
Это показывает authoritative DNS-серверы для домена `ya.ru`.

---

### **Ключевые опции `nslookup`**
- `-query=<тип>`: Запрос определенного типа DNS-записи (например, `A`, `MX`, `NS`, `TXT` и т.д.).
  Пример:
  ```bash
  nslookup -query=mx ya.ru
  ```
- `-type=<тип>`: Аналогично `-query`, указывает тип записи.
- `-debug`: Включает отладочный режим, показывая подробную информацию о запросе.
- `-port=<порт>`: Указывает порт DNS-сервера (по умолчанию 53).

---

### **Когда использовать `nslookup`?**
1. **Диагностика проблем с DNS**:
   Если сайт не открывается, проверь, правильно ли разрешается его доменное имя в IP-адрес.

2. **Проверка MX-записей**:
   Если у тебя проблемы с почтой, проверь MX-записи домена.

3. **Проверка NS-записей**:
   Убедись, что DNS-серверы домена настроены правильно.

4. **Обратный DNS-запрос**:
   Если у тебя есть IP-адрес, и ты хочешь узнать, какому домену он принадлежит.

---

### **Альтернативы `nslookup`**
- **`dig`**: Более мощная утилита для работы с DNS, предоставляет больше информации и гибкости.
- **`host`**: Простая альтернатива для разрешения доменных имен и IP-адресов.

---

### **Итог**
`nslookup` — это удобный инструмент для работы с DNS. Он помогает:
- Узнать IP-адрес домена.
- Проверить обратное разрешение (IP → домен).
- Диагностировать проблемы с DNS.
- Проверять MX, NS и другие типы DNS-записей.
